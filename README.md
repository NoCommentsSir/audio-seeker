
## Audioseeker

**Audioseeker** — это сервис для поиска музыкальных треков по их аудиоотрывкам. Система использует алгоритмы аудиофингерпринтинга для идентификации треков даже при наличии шумов или низкого качества записи.

### Технологический стек
- Инфраструктура: Docker, Docker Compose
- Хранилище файлов: MinIO (S3-compatible)
- База данных: PostgreSQL
- Бэкенд: Python (FastAPI)
- Фронтенд: React

### Архитектурные компоненты
1. Инфраструктура  
Развертывается через docker-compose.
- **MinIO**: Объектное хранилище для физического хранения аудиофайлов.Обеспечивает надежность и доступность медиа-контента.
- **PostgreSQL**: Реляционная база данных для хранения метаданных треков и их аудиоотпечатков. Оптимизирована для быстрого поиска похожих векторов.      
При первом запуске автоматически выполняется скрипт /db/create_schema.sql для создания необходимых таблиц и схем.
2. Бэкенд (Backend)  
Отвечает за обработку запросов, работу с данными и алгоритмы поиска.
- **db/**: Описание моделей данных и конфигурация подключений. Сессии настроены для корректной работы как с PostgreSQL, так и с MinIO.
- **scripts/**: Утилиты для импорта и первичного наполнения базы треков.
- **core/**: Ядро проекта. Содержит реализацию алгоритмов генерации отпечатков и логику сравнения аудиопотоков.
- **app/**: Слой API. Обрабатывает HTTP-запросы, валидирует данные и возвращает результаты поиска.
- **test/**: Набор автотестов для проверки логики API.
3. Фронтенд (Frontend)  
Пользовательский интерфейс для взаимодействия с сервисом.
- **src/**: Исходный код клиентской части (компоненты, стили, логика запросов).
- **test/**: Тесты интерфейса и пользовательских сценариев.

### Дополнительно

Для корректной работы необходимо в корне создать файл `.env` со следующим содержимым
PG_USER, PG_PASSWORD, PG_DB, PG_HOST, POSTGRES_PORT, MINIO_API_PORT, MINIO_PORT, MINIO_USER, MINIO_PASSWORD, MINIO_HOST, MINIO_BUCKET_NAME
